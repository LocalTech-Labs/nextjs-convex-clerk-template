# Micro-SaaS Template

## Stack
- **Next.js 16** (App Router, React 19) on **Cloudflare Workers** via @opennextjs/cloudflare
- **Convex** - Real-time serverless backend and database
- **Clerk** - Auth (synced to Convex via webhook)
- **Clerk Billing** - Payments and subscriptions (via Stripe)
- **shadcn/ui** (new-york style) + **Tailwind CSS v4** (OKLCH colors)
- **PostHog** - Analytics and feature flags (optional)
- **Biome** - Lint/format (replaces ESLint + Prettier)
- **Vitest + Playwright** - Unit and E2E testing

## Quick Reference

### Commands
```
bun run dev              # Convex + Next.js dev servers (concurrent)
bun run build            # Production build
bun run check            # IMPORTANT: Run before committing (Biome lint + format)
bun run test             # Unit tests (Vitest)
bun run test:e2e         # E2E tests (Playwright)
bun run analyze          # Bundle analysis (ANALYZE=true next build)
bun run build:worker     # Build for Cloudflare Workers
bun run deploy           # Deploy to Cloudflare Workers
```

### Key Directories
```
app/                     # Next.js pages and layouts (App Router)
  (marketing)/           # Public pages: landing, pricing, legal
  (app)/                 # Authenticated app pages: dashboard
  providers.tsx          # Clerk + Convex + PostHog provider tree
components/
  layout/                # App-level layout (header)
  ui/                    # shadcn/ui components
convex/                  # Backend: schema, queries, mutations, HTTP routes
hooks/                   # Custom React hooks
lib/                     # Utilities (cn() for classNames)
test/                    # Test utilities, mocks, fixtures
e2e/                     # Playwright E2E tests
```

## Gotchas

- IMPORTANT: Never manually edit files in `convex/_generated/`. These are auto-generated by Convex.
- IMPORTANT: Run `bun run check` before committing. Biome enforces tabs + double quotes.
- Clerk webhook endpoint is at `convex/http.ts` path `/clerk-users-webhook`. Clerk user events sync to Convex `users` table.
- `CLERK_JWT_ISSUER_DOMAIN` and `NEXT_PUBLIC_CLERK_FRONTEND_API_URL` are the same URL, used in different contexts (Convex server vs client).
- Convex actions that need Node.js APIs must use `"use node"` directive at top of file.
- shadcn/ui uses `new-york` style. Add components with: `bunx shadcn@latest add <name>`
- PostHog is optional. If env vars are missing, the provider gracefully no-ops.
- Clerk Billing handles payments. Configure plans in the Clerk dashboard, not in code. Plan data syncs to Convex via the existing Clerk user webhook.
- Route groups `(marketing)` and `(app)` share the root layout but have separate sub-layouts.
- Tailwind v4 uses CSS-based config in `globals.css`, not `tailwind.config.ts`.

## New Feature Checklist

1. **Schema** - Define table in `convex/schema.ts` with typed validators (`v.string()`, etc.)
2. **Backend** - Add queries/mutations in `convex/`. Use `query()`, `mutation()`, `action()` from `"./_generated/server"`
3. **Page** - Create route in `app/(app)/` for authenticated or `app/(marketing)/` for public
4. **Components** - Use shadcn/ui components from `components/ui/`. Merge classes with `cn()`
5. **Auth** - Wrap with `<Authenticated>`/`<Unauthenticated>` from `convex/react`
6. **SEO** - Export `metadata` object from page/layout files
7. **Test** - Add unit tests in `__tests__/` dirs, E2E in `e2e/`
8. **Check** - Run `bun run check` and `bun run test`

## Conventions

- **Formatting**: Biome handles everything. Tabs, double quotes, auto-sorted imports.
- **Imports**: Use `@/` path alias for all project imports.
- **Files**: kebab-case. Components: PascalCase. Functions/variables: camelCase.
- **Tests**: Co-locate in `__tests__/` directories. Use custom render from `@/test/utils/test-utils`.
- **Convex validators**: Always use typed validators from `convex/values` for function args.

## Architecture Decisions

- **Route groups** `(marketing)` / `(app)`: Separate layouts for public vs authenticated sections without affecting URL paths.
- **Convex + Clerk sync**: Clerk webhook pushes user events to Convex HTTP endpoint, keeping a `users` table in sync. This allows querying user data in Convex without extra API calls.
- **Clerk Billing**: Plans and subscriptions are managed in the Clerk dashboard. Billing state syncs to Convex through Clerk user webhooks (plan info stored in user.public_metadata). Frontend uses `<PricingTable />` from @clerk/nextjs and `has({ plan: "pro" })` for feature gating.
- **Cloudflare Workers**: Uses @opennextjs/cloudflare adapter. Build with `build:worker`, deploy with `deploy`.

## Environment Variables

### Required - Convex
- `NEXT_PUBLIC_CONVEX_URL` - Convex deployment URL

### Required - Clerk
- `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` - Frontend key
- `CLERK_SECRET_KEY` - Backend secret
- `NEXT_PUBLIC_CLERK_FRONTEND_API_URL` - Issuer URL (client)
- `CLERK_JWT_ISSUER_DOMAIN` - Issuer URL (Convex auth)
- `CLERK_WEBHOOK_SECRET` - Webhook signing secret

### Optional - PostHog
- `NEXT_PUBLIC_POSTHOG_KEY` - Project API key
- `NEXT_PUBLIC_POSTHOG_HOST` - API host (default: us.i.posthog.com)

See `.env.example` for all variables with descriptions.
